Kingshot Central market validation notes

Research snapshot
- Kingshot-specific search results are dominated by codes, redeem instructions, and isolated guide pages.
- Mature Whiteout Survival sites show the stronger long-term pattern: calculators, alliance tools, progress saving, verified data, trust pages, and community correction loops.
- Reddit threads show players ask for practical beginner routes, alliance choice, hero/formations, event spending advice, and actual useful information rather than generic articles.

Competitor observations
- KingshotGuides.com has an active gift-code page with expiry dates and redeem steps, plus comments where players ask iOS/Android redeem questions.
- Kingshot.net Wiki has deeper hero gear and widget content, including clear priority order and update dates.
- WOS Hub and WoS Tools position themselves around calculators, verified data, local/account saving, and complete upgrade planning.
- WoS Club and WOS Nexus show there is demand for alliance management, hero gear calculators, resource trackers, and redeem workflows.
- Frost Command separates calculators, guides, references, trust pages, and contact support, which is a strong structure for a player utility site.

Current Kingshot Central strengths
- The site already covers daily repeat loops: codes, reset watch, hold/spend decision, hero pull helper, building planner, alliance checklist, local notes, and source/contact pages.
- It has an automated update script and browser refresh for live-feed.json.
- It now has trust basics: ads.txt, robots.txt, sitemap.xml, privacy policy, sources/contact page, contact email, structured data, and fan-made disclaimer.

Gaps to prioritize after launch
- Add a real hero gear calculator: target level, gear type, stat priority, estimated materials, and saved local progress.
- Add a building prerequisite table or calculator using verified player screenshots and reports.
- Add an event prep score sheet for Hall of Heroes / Hero Roulette / alliance scoring windows.
- Add a public "report a code" flow using email or a simple static form provider.
- Add compact share cards for X/Reddit/Discord posts whenever a new code is confirmed.
- Add a small changelog module to the homepage that displays the last update time and exact source.

14-day validation loop
- Day 1: Launch with homepage, codes, today, hero, event, building, alliance, sources/contact, and privacy pages.
- Day 1-3: Share only in context: answer Reddit/Discord questions with a useful tip first, then link the exact page if it solves the question.
- Day 3: Check Search Console indexing, Bing Webmaster indexing, and AdSense policy status.
- Day 4-7: Track which page gets direct visits or emails: codes, today, building planner, or alliance checklist.
- Day 7: Add the first missing high-demand feature based on actual clicks or feedback, likely hero gear or building prerequisites.
- Day 10: Publish one short update post: confirmed codes, expired codes, what changed, and one player tip.
- Day 14: Decide whether to deepen Kingshot Central or clone the structure to the next SLG game based on indexed pages, direct visits, and feedback.

Free promotion rules
- Do not spam a generic homepage link. Share exact utility pages only when they answer a real question.
- Lead with the useful answer in the post, then link as the "full board" or "saved checklist."
- For Reddit, ask for corrections and feedback openly. Communities punish fake authority but often help improve free tools.
- For Discord, alliance checklist and code board are easier to share than broad guide pages.
- For X/Twitter, short code posts work best: code, expiry, redeem method, source confidence, and link.
